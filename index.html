Essential Requirements

OOPs File I/O C++

Project Overview

The Hotel Management System is a console-based C++ application designed to automate and simplify various tasks involved in hotel operations. This system allows hotel staff to book rooms, manage customer details, display allotted room information, edit existing customer records, and even handle restaurant food orders. The program categorizes rooms into Deluxe, Executive, and Presidential types, each with a different pricing model.

Technical Challenges

Challenges included managing binary file operations, preventing data loss during edits, and handling user input validation. Formatting console output neatly and tracking room availability also required careful implementation and testing.

Results

The project successfully automates key hotel functions like booking, checkouts, and billing. It stores data persistently using binary files and offers a simple, user-friendly interface. The system improves efficiency and reduces manual errors.It provides accurate billing based on room type and food services. Additionally, it helps hotel staff manage customer records with ease and speed.

Key Learnings

• OOPs: Learned encapsulation by organizing hotel data and functions within a class structure.
• File I/O: Understood binary file handling for storing and updating customer records efficiently.
• Importance of planning class methods and attributes.
• Gained insight into structuring a console application with a clear user interface.

Sudoku

HTML CSS JavaScript

Project Overview

This project is a fully functional Sudoku game developed using HTML, CSS, and JavaScript. It provides an interactive user interface with features such as a timer, hints, puzzle validation, and auto-solving. The aim was to create a clean, engaging puzzle experience that runs entirely in the browser, with a responsive design and intuitive controls for all levels of users.

Technical Challenges

One of the main challenges was implementing an efficient Sudoku-solving algorithm while keeping the logic modular and readable. Ensuring real-time input validation and integrating features like hints and "check" functionality required managing the game state accurately. Another key challenge was syncing the visual updates with internal logic to reflect user interactions smoothly.

Results

Successfully developed a fully playable Sudoku game featuring a functional and user-friendly interface. The game includes core logic for solving puzzles, providing hints, and validating user inputs. Responsive design principles were applied to ensure smooth interactivity and real-time feedback across different screen sizes. Enhancements like a countdown timer and limited hints were implemented to boost user engagement. The game runs efficiently in modern browsers, with a clean separation between the logic and layout, ensuring maintainability and scalability.

Key Learnings

• Gained practical experience in grid-based layout design and state management in JavaScript.
• Learned to apply backtracking algorithms to solve constraint-based problems like Sudoku.
• Improved skills in event-driven programming, modular coding, and debugging complex UI behavior.
• Understood how to balance user experience (UX) with algorithmic performance in a web environment.

Amazon Clone

HTML CSS

Project Overview

This project is a static Amazon homepage clone developed using HTML and CSS. The design replicates the layout, navigation bar, product cards, and promotional banners of the original Amazon site. It showcases various categories like Garments, Electronics, Wooden Sets, and Health & Personal Care, giving users a feel of browsing a real e-commerce platform.

Technical Challenges

The main challenge was accurately recreating the complex grid layout and responsive design without JavaScript. Managing the positioning and spacing of multiple sections, implementing a flexible navigation bar, and maintaining design consistency across different screen sizes required precision in CSS styling and media queries.

Results

The project successfully resulted in a visually appealing and structurally accurate clone of the Amazon homepage. It effectively demonstrates key design elements such as a responsive navigation bar, product category sections, and promotional banners, all built using only HTML and CSS. The layout adapts well across different screen sizes, showcasing the project's responsive design capabilities. Overall, the clone delivers a realistic front-end experience and serves as a strong foundation for future enhancements using JavaScript or backend integration.

Key Learnings

• Learned how to structure large-scale HTML pages with multiple content sections.
• Strengthened understanding of CSS layout systems, spacing, and alignment techniques.
• Understood the importance of design consistency and UI hierarchy in e-commerce websites.
• Built confidence in replicating professional-grade interfaces using only front-end technologies.
